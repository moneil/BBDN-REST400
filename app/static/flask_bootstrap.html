Flask and Bootstrap

Two best practices for RESTful development are stateless and responsive design. This demo uses two open projects and their libraries to achieve both - Flask and Bootstrap.

Flask and Bootstrap

Flask

Flask is a modular MVC framework - there are other popular frameworks such as Django. Flask was chosen for its modulararity and simplicity. Flask (Controller) uses a template renderer called Jinga for delivery of pages (Views) populated with REST requested data (Model).
For a complete description on Flask and it's usage you may visit

More on Flask

Bootstrap

Bootstrap is a responsive design library originally developed by Twitter which provides the CSS libraries which facilitate building of responsive web pages. There are many design tools available to assist with responsive page layout - licensed and free. This demo was built using Pingendo.
For a comprehensive description of Bootstrap and it's use you may visit the Bootstrap site at http://getbootstrap.com

More on Bootstrap

Bootstrap is built to work best in the latest desktop and mobile browsers, meaning older browsers might display differently styled, though fully functional, renderings of certain components.

More Flask

You may use this demo as a Flask project template, but there are others available - a quick search of the internet will surface a few.
The general functional layout of a Flask based project is as follows: 

 1 project_directory/
 2  __init__.py
 3  run.py
 4  config.py
 4  requirements.txt
 5  app/
 6    static/
 7    templates/
 8    views.py
 9    models...
where run.py is the flask application startup script, config.py contains the application configuration, requirements.txt is a file containing a list of dependencies, the app directory contains the actual application. The static and templates directories contain the static html and jinga templates respectively. The app/views.py is the 'control' file flask uses to direct incoming requests, and there may be multiple models called by the control code.
There is a degree of flexibility in how you organize flask projects including the use of blueprints for more complicated projects.
